# 报销单补件催办与归档工作台

一个完整的报销单审批流转工作台，支持**待审核 → 待补件 → 待复核 → 已通过 → 已驳回 → 已归档**全流程，包含补件催办、权限控制、操作留痕、归档导出等核心能力。

## ✨ 核心特性

- **6 种状态流转**：待审核 / 待补件 / 待复核 / 已通过 / 已驳回 / 已归档
- **4 种角色权限**：申请人 / 审核员 / 财务复核员 / 归档员，各司其职
- **补件催办**：同一补件周期内多次催办**合并记录**，不产生重复历史，保留首次催办时间
- **缺失附件校验**：仍有缺失附件时**不允许审批通过**
- **完整留痕**：所有操作日志、催办记录、驳回原因全部保留
- **逾期提醒**：补件超期自动标记红色闪烁
- **归档导出**：已归档单据可一键导出 JSON 归档包
- **数据持久化**：基于 JSON 文件，重启后数据完全一致
- **预置样例数据**：6 张不同状态的报销单，开箱即用

## 🚀 快速开始

### 1. 安装依赖

```bash
npm install
```

### 2. 导入样例数据

```bash
npm run seed
```

导入后预置 6 张报销单：

| 单号 | 状态 | 说明 |
|------|------|------|
| BX1001 | 待审核 | 上海出差差旅费，含机票+酒店发票 |
| BX1002 | 待补件（已逾期） | 办公用品采购，缺发票+入库单 |
| BX1003 | 待复核 | 员工培训费用，初审已过 |
| BX1004 | 已驳回 | 客户招待费，缺招待名单 |
| BX1005 | 已通过 | 通讯补贴，财务已复核 |
| BX1006 | 已归档 | 交通补贴，已完成归档 |

### 3. 启动服务

```bash
npm start
```

访问 http://localhost:3000

## 📖 主流程实操（推荐按顺序体验）

### 流程：发现缺件 → 发起补件 → 申请人重提 → 财务复核通过 → 归档导出

#### 第 1 步：审核员发现缺件，发起补件

1. 右上角切换到 **李四（审核员）**
2. 左侧点击 **待审核**，选择 **BX1001 上海出差差旅费**
3. 查看附件：有机票和酒店发票，但缺**出差审批单**
4. 点击底部 **📨 发起补件** 按钮
5. 勾选 **审批单**（或手动输入"出差审批单"），设置期限 3 天
6. 点击 **发起补件**

✅ 验证点：
- 状态变为 **待补件**
- 出现 **补件信息** 卡片，显示截止时间、补件轮次
- 附件清单中红色标记 **出差审批单（缺失）**
- **催办记录** 中出现第 1 条记录
- **操作日志** 记录了"发起补件"操作

#### 第 2 步：多次催办，验证同一周期不产生重复记录

1. 保持审核员身份，点击 **⏰ 再次催办** 按钮 2~3 次
2. 查看 **催办记录**

✅ 验证点：
- 催办记录**始终只有 1 条**（同一补件周期合并）
- 催办次数从 1 增加到 3/4 次
- **首次催办时间保持不变**
- **操作日志** 中每次催办都有记录，但标注"同一补件周期，历史合并"

#### 第 3 步：申请人补充材料，重新提交

1. 右上角切换到 **张三（申请人）**
2. 左侧点击 **待补件**，选择 **BX1001**
3. 点击 **📎 提交补件** 按钮
4. 添加附件：名称填 "出差审批单.pdf"，类别选 "审批单"
5. 点击 **提交补件**

✅ 验证点：
- 状态变为 **待复核**
- 附件清单中出差审批单从红色缺失变为正常附件
- 操作日志记录"提交补件"

#### 第 4 步：财务复核通过

1. 右上角切换到 **王五（财务复核员）**
2. 左侧点击 **待复核**，选择 **BX1001**
3. 确认附件齐全，点击 **✅ 复核通过**

✅ 验证点：
- 状态变为 **已通过**
- 操作日志记录"财务复核通过"

#### 第 5 步：归档员归档并导出

1. 右上角切换到 **赵六（归档员）**
2. 左侧点击 **已通过**，选择 **BX1001**
3. 点击 **📦 归档** 按钮
4. 归档后，点击 **⬇️ 导出归档**

✅ 验证点：
- 状态变为 **已归档**
- 下载的 JSON 文件包含：报销单详情 + 催办记录 + 完整操作日志
- 操作日志记录"已归档"

## ❌ 失败链路验证

### 验证 1：缺失附件时不能审批通过

**操作步骤：**
1. 切换到 **王五（财务复核员）**
2. 找到 **BX1002 办公用品采购**（当前待补件状态）
3. 尝试用 API 或前端操作看能否直接通过
4. 或者先让申请人只补一部分，再试

**预期结果：**
- 待补件状态下，没有"通过"按钮
- 即便通过 API 强制调用，也会返回错误：`仍有缺失附件未补充，不允许审批通过`

### 验证 2：申请人无权归档

**操作步骤：**
1. 切换到 **张三（申请人）**
2. 左侧点击 **已通过**，找到 **BX1005 通讯补贴**
3. 查看底部操作区

**预期结果：**
- 操作区显示"当前角色无可用操作"
- 没有"归档"按钮
- 用 API 调用会返回错误：`无权限执行此操作`

### 验证 3：同一催办周期重复提醒不产生多条历史

**操作步骤：**
1. 切换到 **李四（审核员）**
2. 左侧点击 **待补件**，选择 **BX1002**
3. 记录当前催办记录数和首次催办时间
4. 连续点击 **⏰ 再次催办** 3 次
5. 再次查看催办记录

**预期结果：**
- 催办记录数**保持 1 条不变**
- 催办次数从 2 增加到 5
- **首次催办时间保持原样**（不会被覆盖）
- 操作日志中每次都有记录，标注"同一补件周期，历史合并"

### 验证 4：审核员不能越权复核

**操作步骤：**
1. 切换到 **李四（审核员）**
2. 左侧点击 **待复核**，选择 **BX1003**
3. 查看底部操作按钮

**预期结果：**
- 没有"通过"按钮（只有财务才能复核）
- 只有"发起补件"和"驳回"按钮

## 🔧 API 列表

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/meta | 获取元数据（用户、状态、角色） |
| GET | /api/reimbursements | 获取报销单列表 |
| GET | /api/reimbursements/:id | 获取报销单详情（含催办+日志） |
| POST | /api/reimbursements | 创建报销单 |
| POST | /api/reimbursements/:id/approve | 审批通过 |
| POST | /api/reimbursements/:id/reject | 驳回 |
| POST | /api/reimbursements/:id/request-supplement | 发起补件 |
| POST | /api/reimbursements/:id/remind | 再次催办 |
| POST | /api/reimbursements/:id/submit-supplement | 提交补件 |
| POST | /api/reimbursements/:id/archive | 归档 |
| GET | /api/reimbursements/:id/export | 导出归档 |

所有 API 通过 `X-User-Id` 请求头指定当前用户。
